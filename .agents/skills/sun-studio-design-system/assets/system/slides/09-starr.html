Kỹ thuật phỏng vấn

Kỹ thuật "bóc hành" - Peeling the onion

Peel timeline: hỏi sâu từng lớp
S
Situation
Bối cảnh cụ thể xảy ra ở đâu, khi nào?
T
Task
Mục tiêu / trách nhiệm của bạn là gì?
A
Action
Bạn đã làm gì, bằng tool/code/quy trình nào?
R
Reason
Vì sao chọn hướng đó thay vì cách khác?
R
Result
Kết quả đo được và bài học là gì?
Ví dụ thực tế:
“Em đã optimize game performance”
Dừng ở đây = câu trả lời mơ hồ
Bóc tiếp:
S:“Game bị lag ở đâu? Khi nào phát hiện?”
T:“Mục tiêu cải thiện là gì? FPS bao nhiêu?”
A:“Bạn làm gì cụ thể? Code nào? Tool gì?”
R:“Tại sao chọn approach đó thay vì cách khác?”
R:“Kết quả? FPS tăng bao nhiêu? User feedback?”
Càng đào sâu, càng thấy rõ năng lực thực sự
