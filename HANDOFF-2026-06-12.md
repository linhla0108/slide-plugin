## 2. Đã build gì (tất cả đã commit-ready, suite PASS)

### Gate mới — chặn B11

- Đã chạy validator trên v2 cũ → 12 failure, exit 1 (chứng minh gate bắt đúng).

### Script decompose mới — fix B11 ở khâu build deck
- **`slide-system/scripts/decompose_svg_objects.py`** (entry point) +
  **`slide-system/scripts/measure_svg_groups.js`** (helper node/Playwright).
- Cách dùng:
  ```bash
  .venv/bin/python3 slide-system/scripts/decompose_svg_objects.py \
    --svg <item>/artifact/visual.svg \
    --out-dir <run>/assets/page-NN \
    --prefix page-NN --href-base assets/page-NN
  ```
- Pipeline bên trong:
  1. Đo bbox từng group bằng Chromium thật (`getBoundingClientRect`) — SVG
     extraction có matrix transform trên mọi path nên KHÔNG split tĩnh được.
  2. **Tách nền full-bleed ra TRƯỚC** khi cluster (≥85% canvas → `base_candidates`,
     dùng làm CSS background của slide, KHÔNG tag) — thiếu bước này nền nuốt
     cả trang vào 1 cluster (đã dính trên page-01).
  3. Cluster các group **liên tiếp** chồng bbox = 1 object (painter's order giữ
     các mảnh của 1 object cạnh nhau: card = gradient + shadow + mặt trắng).
     KHÔNG dùng all-pairs overlap (mũi tên lọt vào bbox bóng chéo của card → over-merge).
  4. **Explode**: group nguồn trải ≥50% canvas có 2–16 cụm con rời nhau → tách
     theo bbox con (page-03 có TẤT CẢ mũi tên trong 1 `<g>` rộng 1747px).
  5. `--merge-gap` mặc định **0 và là load-bearing**: bóng card-1/card-2 trên
     page-02 chỉ cách nhau 4.08px — gap dương nào cũng làm dính card. ĐỪNG tăng.
- Output: `<prefix>-obj-NN.svg` (fragment, viewBox = bbox + 3px, full defs),
  `snippet.html` (div đã tag sẵn để paste vào slide), `decompose-manifest.json`.
