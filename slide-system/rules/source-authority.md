# Source Authority

Use this order when sources disagree:

1. Explicit user instructions for the current job.
2. Approved job requirements and approval package.
3. Supplied source files for content and reference appearance.
4. Selected brand-pack rules and canonical assets.
5. Published visual-library items.
6. Project-local implementations.

Record every intentional override in the run manifest. Never silently replace
approved source content or treat a prototype as the canonical design system.
When supplied source formats disagree, create a divergence summary and ask the
user which source is authoritative. Do not infer authority from timestamps.

For SUN.STUDIO, canonical brand assets remain under
`.agents/skills/sun-studio-design-system/assets/system/`.
