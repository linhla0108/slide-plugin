# Approval Gates

## Slide Generation

Use one explicit approval gate before build. The package includes requirements,
capabilities, content plan, visual plan, selections and rejections, export
strategy, risks, limitations, and requested overrides.

Build must not start until approval is recorded in the run manifest.

## Component Extraction

Extraction is manual-only. The request must identify source path, slide or page,
and exact region or object. Each extracted item remains in staging until the
user explicitly approves publication.

Approval for one extraction item does not approve other items in the batch.
